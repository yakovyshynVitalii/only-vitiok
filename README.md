# only-vitiok

Автоматизований пайплайн для підготовки та завантаження медіа з AI-генерацією metadata.

## Можливості

1. Аналізує фото/відео з папки `media`.
2. Генерує `media-config.json`:
   - `hashtags`: до 15 глобальних хештегів.
   - `items`: масив файлів з `title`, `description`, `hashtags`, `vip`, `filePath`.
3. Автоматично заповнює metadata на сторінці завантаження.
4. Підтримує паралельний AI-аналіз (швидше на великих наборах файлів).

## Як це працює

1. `scripts/generate-config.js` бере медіа з `MEDIA_FOLDER`.
2. Для кожного файлу запитує Ollama (з урахуванням `AI_ANALYSIS_CONCURRENCY`).
3. Нормалізує текст, фільтрує теги через `tag-categories.txt`, збирає фінальний `media-config.json`.
4. `upload.js` завантажує файли й заповнює `title/description/hashtags`.

## Швидкий старт

1. Встанови залежності:

```bash
npm install
```

2. Створи `.env` на основі `.env.example`.
3. Підготуй модель Ollama:

```bash
ollama pull qwen2.5vl:3b
ollama list
```

4. Увійди в акаунт (збереже сесію для Playwright):

```bash
node login.js
```

5. Згенеруй конфіг:

```bash
npm run generate-config
```

6. Додай/онови теги в інтерфейсі (опціонально):

```bash
npm run add-tags
```

7. Запусти аплоад:

```bash
npm run upload
```

Повний пайплайн однією командою:

```bash
npm run vitiok
```

## Конфігурація `.env`

Обов'язкові ключі:

- `OLLAMA_URL` - URL локального Ollama API (приклад: `http://127.0.0.1:11434`).
- `BASE_URL` - базовий URL сайту.
- `CREATE_URL` - URL сторінки створення/аплоаду.

Рекомендовані ключі:

- `OLLAMA_MODEL` - модель для аналізу (типово: `llava:13b`).
- `AI_ANALYSIS_CONCURRENCY` - кількість одночасних AI-запитів (типово: `3`, безпечний діапазон: `3-5`).
- `MEDIA_FOLDER` - папка з файлами (типово: `./media`).
- `MEDIA_CONFIG_PATH` - шлях для збереження конфіга (типово: `./media-config.json`).
- `TAG_CATEGORIES_PATH` - файл дозволених тегів (типово: `./tag-categories.txt`).

## Теги

- Редагуй дозволені теги у `tag-categories.txt`.
- Генератор використовує тільки теги з цього файлу.
- Для кожного медіа зберігаються індивідуальні теги, а глобальні використовуються як доповнення.

## Продуктивність

- Для відео бажано встановити `ffmpeg`/`ffprobe` (береться кадр для аналізу).
- Для зображень на macOS використовується `sips` для ресайзу перед AI-аналізом.
- Якщо Ollama тимчасово недоступний, скрипт створить fallback metadata і продовжить обробку.

## Діагностика

- Повільно працює: зменш/збільш `AI_ANALYSIS_CONCURRENCY` (почни з `3`, далі `4-5`).
- Дивні теги: перевір `tag-categories.txt` та якість вихідних файлів.
- Помилка підключення: перевір, що `ollama serve` запущений і `OLLAMA_URL` правильний.
